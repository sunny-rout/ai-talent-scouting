👩‍💼 Recruiter Input

The recruiter pastes any free-text Job Description into a textarea. No special format required — the LLM handles structure extraction in the next step.

3 sample JDs pre-loaded:

{% for role, color in [("Software Engineer","#e8f4f3"),("Data Scientist","#fef3c7"),("Product Manager","#f0e8ff")] %}
{{ role }}
{% endfor %}
Output contract: raw jd_text: str posted to POST /parse-jd

🔍 JD Parser

Single LLM call with temperature=0.2 for deterministic JSON output. Falls back to regex-based extraction if the LLM returns malformed JSON.

Prompt strategy:
system: "Return only valid JSON"
user: schema + JD text
temp: 0.2 (low = consistent)
Output contract: ParsedJD Pydantic model with 8 fields
{% for tag in ["role","role_type","required_skills","preferred_skills","years_experience","education","must_haves","salary_range"] %} {{ tag }} {% endfor %}

🎯 Candidate Matcher

Pure rule-based scoring — fast, deterministic, explainable. No LLM needed. Fuzzy skill normalization strips spaces, hyphens, and case.

🏆 Final Ranker

Weighted combination of both scores. Match has higher weight (60%) because it is objective and verifiable. Interest (40%) differentiates equally skilled candidates.

Final Score
= 0.6 × Match Score
+ 0.4 × Interest Score
